What is Responsive Redesign?

Have you ever gone through a new website and noticed flaws in the interface?
Today we will grow through the process of redesigning a website by creating low-fidelity and high-fidelity prototypes for various screen sizes and building a responsive website based on those prototypes.

Website: Yale School of Art

The Yale School of Art website is composed of distracting animated backgrounds, inconsistent use of colors, font styles, layout design, and so much more that creates a negative user experience.

Link: Yale School of Art

I recognize the fact that this is a colaborative experiment where those with a Yale ID can edit the webpage. I am disregarding that aspect and focusing solely on the design and usability of the website itself.

Usability & Accessibility Problems

The following are notes on the usability and accessbility issues with this interface.

Usability:

  • It is a completely overwhelming interface.
  • There is no consistent style or color palette.
  • There is no hierarchy; there's a need to look everywhere to locate something or to make sure nothing important is missed.
  • The website almost feels fake and untrustworthy due to the design. At first site, someone might have thought they clicked on the wrong link in order to get to the Yale School of Art.
  • The purpose of this site is to apply to the school, however the link to start an application is not clear.
  • Learnability: Because it is very distracting, it forces users to get familiar with the whole interface.
  • Memorability: Several different styles and colors is hard to remember, especially for casual users.
  • Efficiency: It is a very unique looking website, so a frequent user might get used to looking at distinct aspects of the website to locate where they need to go.
Accessibility:
  • After running it through the WAVE tool to check for accessibility problems, no errors or contrast errors appeared. However, I completely disagree.
  • Some of the text, including the address at the top and the contact options below, is too small to read.
  • The highlighting behind the text can make it difficult to read as well, especially since there is no padding around the words.
  • Some of the background colors and text colors do not work well with one another, providing a poor contrast for users with visual impairments.
  • As for a user who needs to navigate via keyboard or audio, there are some alerts with missing heading levels or skipped headings to could potentially cause issues.

High-Fidelity Wireframing


    Notes on Redesign:
  1. The main information, application information, and important dates boxes utilize flex to allow the sections to reorder when the screen is made smaller.
  2. When you hover over the options in the navigation bar, they will change a different color to signal the user that it is clickable.
  3. The “Apply Here” buttons, the buttons in the “More Information” section, and the links in the footer will make your cursor change into a pointer when you hover over it.
  4. All the text remains relatively the same size, with some slight movement from the sections and the buttons when the screen size is adjusted.
